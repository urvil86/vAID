# V-Aid — Docs

Project documentation and prompt docs live here.

Drop your prompt specs, build specs, and design notes in this folder (Markdown
`.md` is easiest to read on GitHub, but any format is fine).

Suggested layout:

- `docs/prompts/` — prompt engineering docs, model instructions, question banks
- `docs/specs/` — build/product specifications
- `docs/notes/` — design decisions, meeting notes, misc

Nothing here is imported by the app — it's reference material for the team, so
you can add, rename, or reorganize freely without affecting the build.
